# E-CommerXE Backend

Multi-tenant digital business advertising platform — NestJS v10 · PostgreSQL 15 · Redis 7 · Cloudflare R2

---

## Architecture

```
┌─────────────────────────────────────────────────────┐
│          Nginx (Rate Limiting · WAF · TLS)           │
└────────────────────────┬────────────────────────────┘
                         │
        ┌────────────────┼────────────────┐
        ▼                ▼                ▼
   ┌─────────┐     ┌──────────┐    ┌──────────┐
   │  Auth   │     │ Business │    │  Search  │
   │ Module  │     │  Module  │    │ (PG FTS) │
   └─────────┘     └──────────┘    └──────────┘
        │                │
   ┌────▼────────────────▼────────────────┐
   │       PostgreSQL 15 (RLS enabled)     │
   │   Redis 7 (Cache · Queue · Sessions)  │
   │   Cloudflare R2 (Media Storage)       │
   └────────────────────────────────────-─┘
```

**Tenant isolation** is enforced at the database layer via PostgreSQL Row-Level Security. Every query sets `app.current_tenant` via `SET LOCAL` inside a transaction — no developer mistake can leak cross-tenant data.

## Quick Start

### 1. Prerequisites

- Node.js 20 LTS
- Docker + Docker Compose
- OpenSSL (for key generation)

### 2. Generate RSA keys

```bash
chmod +x scripts/generate-keys.sh
./scripts/generate-keys.sh
```

### 3. Configure environment

```bash
cp .env.example .env
# Edit .env — fill in DB, Redis, R2, SMTP, Google OAuth credentials
```

### 4. Start the stack

```bash
docker compose up -d
```

### 5. Run migrations

```bash
npm install
./scripts/migrate.sh run
```

### 6. Start the API (dev mode with hot-reload)

```bash
npm run start:dev
```

- API: http://localhost:3000
- Swagger docs: http://localhost:3000/api/docs
- Grafana: http://localhost:3001
- Prometheus: http://localhost:9090

## Environment Variables

See `.env.example` for the full list with descriptions. Key variables:

| Variable | Description |
|---|---|
| `JWT_PRIVATE_KEY_PATH` | Path to RS256 private key (generated by `generate-keys.sh`) |
| `JWT_PUBLIC_KEY_PATH` | Path to RS256 public key |
| `JWT_REFRESH_SECRET` | HS256 secret for refresh tokens (min 32 chars) |
| `R2_ACCOUNT_ID` | Cloudflare account ID for R2 storage |
| `TURNSTILE_SECRET_KEY` | Cloudflare Turnstile server-side secret |
| `ENCRYPTION_KEY` | 32-char AES-256 key for at-rest encryption |

## API Reference

All responses are wrapped in a standard envelope:

```json
{
  "success": true,
  "data": { ... },
  "meta": { "requestId": "uuid", "timestamp": "ISO8601" }
}
```

Errors return:

```json
{
  "statusCode": 400,
  "error": "Bad Request",
  "message": "Validation failed",
  "path": "/api/v1/auth/register",
  "timestamp": "ISO8601"
}
```

## Security

- **Passwords**: Argon2id (memoryCost=64MB, timeCost=3, parallelism=4)
- **JWT**: RS256 asymmetric (access 15min) + HS256 refresh (7 days, rotated)
- **Account lockout**: 5 failed attempts → 15min lockout + email notification
- **Captcha**: Cloudflare Turnstile on register endpoint
- **Rate limiting**: Nginx zones per route + NestJS Throttler as fallback
- **RLS**: PostgreSQL Row-Level Security on all tenant tables
- **Media**: Magic byte validation + ClamAV async scan + EXIF strip
- **Headers**: HSTS 2yr, CSP, X-Frame-Options DENY, Referrer-Policy
- **Secrets**: All config via env vars — zero hardcoded credentials
